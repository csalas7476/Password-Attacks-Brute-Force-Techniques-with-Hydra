# Password-Attacks-Brute-Force-Techniques-with-Hydra
Performed brute-force password testing using Hydra against a locally hosted OWASP Juice Shop application, while analyzing intercepted traffic in Burp Suite to evaluate authentication security and response behavior.

1. Set up the testing environment  

2. Deployed OWASP Juice Shop  
   - Pulled the Juice Shop image from Docker Hub  
   - Ran the container on a local port (3000:3000)  

3. Configured Burp Suite  
   - Launched Burp Suite Community Edition  
   - Set up proxy listener (127.0.0.1:8080)  
   - Enabled intercept mode  

4. Connected browser to Burp Suite  
   - Configured browser proxy settings  
   - Installed Burp CA certificate for HTTPS interception  

5. Accessed the web application  
   - Opened OWASP Juice Shop via localhost  
   - Verified traffic was being captured in Burp Suite  

6. Identified the login request  
   - Navigated to the login page  
   - Entered test credentials  
   - Captured the POST request in Burp  
   - Analyzed login parameters (email and password)  

7. Prepared for brute-force testing  
   - Identified the login endpoint and request structure  
   - Selected a password wordlist for testing  

8. Performed brute-force attack using Hydra  
   - Configured Hydra to target the login form  
   - Executed multiple automated login attempts against test accounts  

9. Monitored traffic in Burp Suite  
   - Observed repeated login requests generated by Hydra  
   - Compared failed vs successful responses  
   - Reviewed headers, payloads, and response codes  

10. Analyzed authentication behavior  
    - Evaluated how the application handled multiple login attempts  
    - Checked for rate limiting or account lockout mechanisms  
    - Observed error messages returned by the server  

11. Documented findings  
    - Recorded observations and potential vulnerabilities  
    - Captured screenshots of intercepted traffic
